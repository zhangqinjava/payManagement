## 1.account-service(账户模块)
### 1.1 项目简介
account-service 是一个 **高并发、安全、可扩展的账户资金模块**，  
用于支撑支付系统中的 **余额管理、上账、下账、付款、冻结/解冻、转账、账务流水记录** 等核心能力。
该模块适用于：
- 支付系统
- 清结算系统
- 资金中台
- 风控冻结场景
### 1.2 核心能力
-  账户余额管理（可用余额 / 冻结余额）
-  账户冻结 / 解冻（支持部分冻结、部分解冻）
-  账户转账（双账户一致性）
-  账务流水 & 明细记录
-  高并发下资金安全保证
-  支持分布式部署
### 1.3 账户资金模型
#### 1.3.1 账户字段说明
```text
balance         账户账面余额（不直接减少）
frozen_balance  冻结金额
available       可用余额 = balance - frozen_balance
```
#### 1.3.2 设计原则
 冻结 ≠ 扣款
 所有资金操作使用 SQL 原子校验
 禁止「先查余额 → 再更新」
### 1.4 并发与一致性设计
- 使用 Redisson 分布式锁
- 多账户操作使用 MultiLock
- 锁在事务开始前获取，事务完成后释放
- 所有资金更新使用 单条 SQL 原子更新
### 1.5 技术栈
- Java 1.8+
- Spring Boot
- MyBatis-Plus
- MySQL
- Redisson
- Lombok
### 1.6 注意事项
- 所有金额字段使用 BigDecimal
- 所有冻结 / 解冻操作必须保证幂等
- 禁止绕过账户模块直接更新账户表

## 2.merchant-module(商户模块) 
### 2.1 项目简介
merchant-service 负责管理商户信息、商户费率、商户账户绑定信息，是订单和支付流程的入口模块
### 2.2 核心能力
- 商户基础信息管理（名称、等级、状态）
- 商户账户管理（绑定的资金账户）
- 商户费率配置（不同业务场景费率）
- 商户终端的管理(绑定业务终端)
- 商户结算周期配置(可配置 D0 D1 T0 T1 SS )
- 商户与订单关联管理
- 商户风控标记与黑名单管理
### 2.3  技术栈
- Java 1.8+
- Spring Boot
- MyBatis-Plus
- MySQL
- Lombok
- redis缓存
### 2.4 注意事项
- 商户费率变更需版本化，历史订单不受影响
- 商户账户必须绑定到 account-service 账户才能进行资金操作
- 接口操作需幂等化，防止重复创建或更新

## 3.order-module (订单模块)
### 3.1 项目简介
order-service 负责订单的生成、支付、退款、状态管理，是支付系统中账户与商户交互的核心模块。
主要功能：
- 订单创建、修改、查询
- 支付状态管理（待支付、支付成功、支付失败）
- 退款状态管理（退款中、已退款）
- 关联商户和资金账户
- 异常订单处理（支付失败、超时关闭）
- 支持分布式事务或消息最终一致性（TCC / MQ）
### 3.2 技术栈
- Java 1.8+
- Spring Boot
- MyBatis-Plus
- MySQL
- Lombok
